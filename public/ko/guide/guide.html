운영체제(OS)란 무엇인가요?
운영체제는 컴퓨터의 두뇌 역할을 하는 핵심 소프트웨어입니다. 운영체제가 하는 일을 쉽게 설명하면, 여러분이 마우스를 클릭하거나 키보드를 누를 때 그 신호를 받아서 적절한 프로그램에 전달하고, 프로그램이 화면에 결과를 보여주도록 합니다. 또한 메모리와 저장공간을 효율적으로 나누어 주고, 여러 프로그램이 동시에 실행될 수 있게 해줍니다. 스마트폰에 안드로이드나 iOS가 있듯이, 컴퓨터에도 윈도우, 맥OS, 리눅스 같은 운영체제가 있습니다. 운영체제 없이는 컴퓨터를 사용할 수 없으며, 운영체제가 다르면 사용법도 조금씩 달라집니다. 컴퓨터를 켜면 가장 먼저 실행되는 것이 바로 운영체제이고, 모든 다른 프로그램들은 운영체제 위에서 동작합니다.
리눅스란 무엇인가요?
리눅스는 전 세계 개발자들이 함께 만들어가는 무료 운영체제입니다. 1991년 핀란드의 대학생 리누스 토르발스가 처음 만들기 시작했고, 지금은 수많은 사람들이 참여해서 개발하고 있습니다. 윈도우는 마이크로소프트라는 한 회사에서 만들고 판매하는 상품이지만, 리눅스는 누구나 무료로 사용할 수 있고, 심지어 내부 구조까지 공개되어 있습니다. 이는 마치 레시피가 공개된 요리와 비밀 레시피로 만든 요리의 차이와 같습니다. 윈도우는 사용자가 쉽게 사용할 수 있도록 그래픽 인터페이스에 중점을 두었다면, 리눅스는 안정성과 보안, 그리고 사용자 맞춤형 설정에 강점이 있습니다. 윈도우에서는 마우스 클릭으로 대부분의 작업을 하지만, 리눅스에서는 텍스트 명령어를 직접 입력하는 경우가 많습니다. 리눅스는 서버 컴퓨터에서 특히 인기가 높습니다. 인터넷의 대부분 웹사이트가 리눅스 서버에서 운영되고 있으며, 안드로이드 스마트폰도 실제로는 리눅스 기반입니다. 개발자와 시스템 관리자들이 리눅스를 선호하는 이유는 강력한 기능과 높은 안정성, 그리고 무료라는 점 때문입니다.
GUI vs CLI - 컴퓨터의 사용 방식
GUI와 CLI는 컴퓨터와 소통하는 두 가지 다른 방식입니다. GUI는 그래픽 사용자 인터페이스로, 우리가 일반적으로 컴퓨터를 사용할 때 보는 화면입니다. 마우스로 아이콘을 클릭하고, 창을 열고 닫고, 메뉴를 선택하는 방식입니다. 윈도우의 바탕화면이나 맥의 파인더가 대표적인 GUI입니다. 직관적이고 배우기 쉬워서 컴퓨터 초보자도 쉽게 사용할 수 있습니다. CLI는 명령줄 인터페이스로, 텍스트로 명령어를 직접 입력해서 컴퓨터와 소통하는 방식입니다. 검은 화면에 하얀 글자로 명령어를 타이핑하는 모습을 영화에서 본 적이 있을 겁니다. 처음에는 어려워 보이지만, 익숙해지면 매우 빠르고 정확하게 작업할 수 있습니다. CLI의 장점은 속도와 정확성입니다. 마우스로 여러 번 클릭해야 할 작업을 한 줄의 명령어로 끝낼 수 있고, 같은 작업을 반복할 때도 명령어를 복사해서 쉽게 실행할 수 있습니다. 또한 원격으로 다른 컴퓨터를 관리할 때는 CLI가 훨씬 효율적입니다. 리눅스에서는 GUI와 CLI 모두 지원하기 떄문에 필요한 인터페이스를 고르시면 됩니다.
오픈소스와 자유 소프트웨어의 개념
오픈소스는 소프트웨어의 설계도인 소스코드를 누구나 볼 수 있도록 공개한 것을 말합니다. 마치 요리 레시피를 모든 사람에게 공개하는 것과 같습니다. 이 레시피를 보고 누구나 요리를 만들 수 있고, 더 맛있게 만들기 위해 레시피를 개선할 수도 있습니다. 전통적인 소프트웨어는 회사에서 만들어서 판매하는 상품입니다. 소스코드는 기업의 비밀이고, 사용자는 완성된 프로그램만 구매해서 사용합니다. 하지만 오픈소스 소프트웨어는 소스코드가 공개되어 있어서 누구나 내부 구조를 확인할 수 있고, 필요에 따라 수정하거나 개선할 수 있습니다. 자유 소프트웨어는 오픈소스와 비슷하지만 조금 더 철학적인 개념입니다. 사용자가 소프트웨어를 자유롭게 사용하고, 복사하고, 수정하고, 배포할 수 있는 권리를 강조합니다. 여기서 자유는 무료를 의미하는 것이 아니라, 자유롭게 사용할 수 있는 권리를 의미합니다. 리눅스가 대표적인 오픈소스이자 자유 소프트웨어입니다. 전 세계 수많은 개발자들이 자발적으로 참여해서 함께 개발하고 있으며, 누구나 무료로 사용할 수 있습니다. 이런 협력적인 개발 방식 덕분에 리눅스는 매우 안정적이고 보안이 뛰어납니다. 많은 사람이 코드를 검토하기 때문에 버그나 보안 문제를 빨리 발견하고 해결할 수 있기 때문입니다.
파일 시스템 구조 이해하기
리눅스의 파일 시스템은 거대한 나무와 같은 구조를 가지고 있습니다. 맨 위에 뿌리가 있고, 거기서부터 가지가 뻗어나가는 형태입니다. 이 뿌리를 루트라고 하며, 슬래시 기호 하나로 표현합니다. 루트 아래에는 여러 개의 중요한 폴더들이 있습니다. home 폴더는 각 사용자의 개인 공간입니다. 윈도우의 내 문서 폴더와 비슷한 역할을 합니다. 만약 사용자 이름이 kim이라면, /home/kim 폴더가 그 사용자의 개인 공간이 됩니다. 여기에 개인 파일들과 설정들을 저장합니다. etc 폴더는 시스템 설정 파일들이 모여있는 곳입니다. 컴퓨터가 어떻게 동작해야 하는지에 대한 규칙들이 저장되어 있습니다. 마치 건물의 관리사무소 같은 역할을 합니다. 네트워크 설정, 사용자 정보, 프로그램 설정 등 중요한 시스템 정보들이 여기에 있습니다. bin 폴더에는 기본적인 명령어 프로그램들이 들어있습니다. ls, cp, mv 같은 자주 사용하는 명령어들의 실제 프로그램 파일들이 여기에 저장되어 있습니다. usr 폴더는 사용자가 설치한 프로그램들이 들어가는 곳이고, var 폴더에는 로그 파일처럼 내용이 자주 변하는 파일들이 저장됩니다. 이런 구조화된 시스템 덕분에 파일들을 체계적으로 관리할 수 있고, 어떤 파일이 어디에 있는지 쉽게 찾을 수 있습니다. 처음에는 복잡해 보이지만, 규칙을 이해하면 매우 논리적이고 효율적인 구조라는 것을 알 수 있습니다.
패키지 관리자란 무엇인가요?
패키지 관리자는 소프트웨어를 설치하고 관리해주는 도구입니다. 마치 스마트폰의 앱스토어와 비슷한 역할을 합니다. 윈도우에서는 프로그램을 설치할 때 인터넷에서 설치 파일을 다운로드받아서 실행해야 하지만, 리눅스에서는 패키지 관리자가 이 모든 과정을 자동으로 처리해줍니다. 패키지 관리자의 가장 큰 장점은 의존성 관리입니다. 어떤 프로그램을 실행하려면 다른 프로그램이나 라이브러리가 필요한 경우가 많습니다. 예를 들어, 음악 프로그램을 설치하려면 오디오 처리 라이브러리가 필요할 수 있습니다. 패키지 관리자는 이런 관계를 자동으로 파악해서 필요한 모든 구성요소를 함께 설치해줍니다. 리눅스 배포판마다 다른 패키지 관리자를 사용합니다. 우분투와 데비안에서는 APT를 사용합니다. apt install firefox라고 입력하면 파이어폭스 브라우저가 자동으로 설치됩니다. 레드햇과 센트OS에서는 YUM이나 DNF를 사용하고, 아치 리눅스에서는 Pacman을 사용합니다. 패키지 관리자를 사용하면 소프트웨어 업데이트도 매우 쉽습니다. 한 번의 명령어로 시스템의 모든 프로그램을 최신 버전으로 업데이트할 수 있습니다. 또한 프로그램을 제거할 때도 관련된 모든 파일을 깨끗하게 삭제해주어서 시스템이 깔끔하게 유지됩니다. 이런 편의성 때문에 리눅스 사용자들은 패키지 관리자를 매우 선호합니다.
터미널이란 무엇인가요?
터미널은 컴퓨터와 텍스트로 대화할 수 있는 창구입니다. 검은 화면에 흰 글씨로 명령어를 입력하는 그 화면이 바로 터미널입니다. 마치 컴퓨터와 채팅하는 것처럼 명령어를 입력하면 컴퓨터가 결과를 텍스트로 보여줍니다. 터미널의 역사는 매우 오래되었습니다. 컴퓨터에 마우스나 그래픽 화면이 없던 시절부터 사용되어 온 방식입니다. 그 시절에는 터미널이 컴퓨터를 사용하는 유일한 방법이었습니다. 지금은 그래픽 인터페이스가 발달했지만, 터미널은 여전히 매우 강력하고 유용한 도구로 남아있습니다. 터미널을 사용하는 이유는 여러 가지가 있습니다. 첫째, 속도가 빠릅니다. 마우스로 여러 번 클릭해야 할 작업을 한 줄의 명령어로 끝낼 수 있습니다. 둘째, 정확합니다. 같은 명령어를 반복 실행하면 항상 같은 결과를 얻을 수 있습니다. 셋째, 자동화가 가능합니다. 여러 개의 명령어를 연결해서 복잡한 작업을 자동으로 처리할 수 있습니다. 리눅스에서 터미널은 특히 중요합니다. 시스템 관리, 프로그램 설치, 파일 관리 등 대부분의 고급 작업들이 터미널을 통해 이루어집니다. 처음에는 어렵게 느껴질 수 있지만, 기본적인 명령어들을 익히면 컴퓨터를 훨씬 효율적으로 사용할 수 있습니다. 개발자들이나 시스템 관리자들에게는 없어서는 안 될 필수 도구입니다.
기본적인 CLI 명령어 알아보기
리눅스 터미널에서 자주 사용하는 기본 명령어들을 익히면 컴퓨터를 훨씬 효율적으로 사용할 수 있습니다. 이 명령어들은 파일과 폴더를 관리하는 기본적인 도구들입니다. ls 명령어는 현재 위치의 파일과 폴더 목록을 보여줍니다. 윈도우 탐색기를 열어서 파일들을 보는 것과 같은 역할입니다. ls -l을 입력하면 파일의 자세한 정보까지 함께 볼 수 있습니다. cd 명령어는 폴더를 이동할 때 사용합니다. cd Documents라고 입력하면 Documents 폴더로 이동합니다. cd ..는 상위 폴더로 이동하고, cd만 입력하면 사용자의 홈 폴더로 돌아갑니다. cp는 파일을 복사하는 명령어입니다. cp file1.txt file2.txt라고 입력하면 file1.txt를 복사해서 file2.txt라는 이름으로 만듭니다. mv는 파일을 이동하거나 이름을 바꿀 때 사용합니다. mv old.txt new.txt는 파일 이름을 바꾸고, mv file.txt /home/user/는 파일을 다른 폴더로 이동시킵니다. rm은 파일을 삭제하는 명령어입니다. rm file.txt라고 입력하면 해당 파일이 삭제됩니다. 주의할 점은 리눅스에서는 휴지통이 없어서 삭제된 파일은 복구하기 어렵습니다. mkdir은 새 폴더를 만드는 명령어입니다. mkdir newfolder라고 입력하면 newfolder라는 이름의 폴더가 생성됩니다. sudo는 관리자 권한으로 명령어를 실행할 때 사용합니다. 시스템 파일을 수정하거나 프로그램을 설치할 때 필요합니다. sudo apt install firefox처럼 다른 명령어 앞에 붙여서 사용합니다.
텍스트 편집기 소개
리눅스에서 텍스트 편집기는 매우 중요한 도구입니다. 설정 파일을 수정하거나, 프로그램 코드를 작성하거나, 간단한 메모를 작성할 때 사용합니다. 터미널에서 바로 사용할 수 있는 텍스트 편집기들이 여러 개 있습니다. nano는 가장 사용하기 쉬운 터미널 텍스트 편집기입니다. nano filename.txt라고 입력하면 해당 파일을 편집할 수 있습니다. 화면 아래쪽에 단축키가 표시되어 있어서 초보자도 쉽게 사용할 수 있습니다. Ctrl+X를 누르면 저장하고 종료할 수 있습니다. 간단한 파일 편집에는 nano가 가장 적합합니다. vim은 매우 강력하지만 배우기 어려운 편집기입니다. 전문 개발자들이나 시스템 관리자들이 선호하는 도구입니다. vim의 특징은 키보드만으로 모든 작업을 할 수 있다는 점입니다. 마우스를 전혀 사용하지 않고도 매우 빠르게 텍스트를 편집할 수 있습니다. 하지만 독특한 조작 방식 때문에 익숙해지는데 시간이 걸립니다. vim에는 여러 가지 모드가 있습니다. 처음 실행하면 명령 모드로, 이때는 텍스트를 직접 입력할 수 없고 커서 이동이나 삭제 같은 명령만 할 수 있습니다. i를 누르면 입력 모드로 바뀌어서 텍스트를 입력할 수 있습니다. ESC를 누르면 다시 명령 모드로 돌아갑니다. emacs도 vim과 함께 유명한 텍스트 편집기입니다. vim보다는 조금 더 직관적이지만 여전히 배우기는 어렵습니다. 각 편집기마다 고유한 장점과 특징이 있어서, 사용자의 필요와 선호에 따라 선택하면 됩니다. 초보자라면 nano부터 시작해서 점차 다른 편집기들을 배워가는 것이 좋습니다.
프로세스 관리 방법
프로세스는 현재 실행 중인 프로그램을 의미합니다. 컴퓨터에서 여러 프로그램이 동시에 실행되고 있을 때, 각각의 실행 중인 프로그램이 하나의 프로세스입니다. 리눅스에서는 이런 프로세스들을 관리하고 모니터링할 수 있는 다양한 도구들이 제공됩니다. ps 명령어는 현재 실행 중인 프로세스 목록을 보여줍니다. ps만 입력하면 현재 터미널에서 실행된 프로세스들만 보이지만, ps aux라고 입력하면 시스템의 모든 프로세스를 볼 수 있습니다. 각 프로세스마다 고유한 번호인 PID가 있고, 누가 실행했는지, 언제 시작되었는지, 얼마나 많은 자원을 사용하고 있는지 등의 정보를 확인할 수 있습니다. top 명령어는 실시간으로 프로세스 상태를 보여주는 도구입니다. 마치 윈도우의 작업 관리자와 비슷한 역할을 합니다. 어떤 프로세스가 CPU를 많이 사용하고 있는지, 메모리를 많이 차지하고 있는지 실시간으로 확인할 수 있습니다. q를 누르면 top에서 나올 수 있습니다. htop은 top보다 더 보기 좋고 사용하기 쉬운 버전입니다. 색깔로 구분되어 있고 마우스로도 조작할 수 있어서 더 직관적입니다. 하지만 기본적으로 설치되어 있지 않아서 별도로 설치해야 합니다. kill 명령어는 프로세스를 종료시킬 때 사용합니다. 프로그램이 멈춰서 응답하지 않을 때 강제로 종료시킬 수 있습니다. kill 1234처럼 프로세스의 PID를 지정해서 종료시킵니다. killall firefox처럼 프로그램 이름으로 종료시킬 수도 있습니다. 이런 도구들을 사용하면 시스템의 상태를 파악하고 문제가 생긴 프로그램을 안전하게 종료시킬 수 있습니다.
IP, DNS, 포트 개념 이해하기
네트워크의 세계에서 IP, DNS, 포트는 인터넷 통신의 핵심 요소들입니다. 이들을 이해하면 인터넷이 어떻게 작동하는지 알 수 있습니다. IP 주소는 인터넷상의 각 컴퓨터가 가지는 고유한 주소입니다. 마치 우편물을 보낼 때 사용하는 집 주소와 같습니다. 192.168.1.1처럼 점으로 구분된 네 개의 숫자로 이루어져 있습니다. 인터넷에 연결된 모든 기기는 이런 IP 주소를 가지고 있어서, 이 주소를 통해 서로 통신할 수 있습니다. 집에서 사용하는 공유기도 IP 주소를 가지고 있고, 여러분의 스마트폰이나 노트북도 각각 고유한 IP 주소를 받아서 인터넷을 사용합니다. DNS는 도메인 이름 시스템으로, IP 주소를 사람이 기억하기 쉬운 이름으로 바꿔주는 서비스입니다. siliod.com이나 google.com처럼 우리가 평소에 사용하는 웹사이트 주소가 바로 도메인 이름입니다. 실제로는 구글도 특정 IP 주소를 가지고 있지만, 숫자로 된 IP 주소를 외우기는 어렵기 때문에 DNS가 이를 도메인 이름으로 변환해줍니다. 마치 전화번호부에서 이름으로 전화번호를 찾는 것과 같은 원리입니다. 포트는 하나의 컴퓨터 안에서 여러 서비스를 구분하는 번호입니다. 한 건물에 여러 호실이 있는 것처럼, 하나의 IP 주소를 가진 컴퓨터에서도 여러 서비스가 동시에 실행될 수 있습니다. 웹서버는 보통 80번 포트를, 이메일 서버는 25번 포트를 사용합니다. 웹사이트에 접속할 때 www.example.com:80처럼 포트 번호를 명시할 수 있지만, 웹은 기본적으로 80번 포트를 사용하기 때문에 생략해도 됩니다.
자주 마주치는 에러와 해결 방법
리눅스를 사용하다 보면 다양한 에러 메시지를 만나게 됩니다. 처음에는 당황스럽지만, 대부분의 에러는 몇 가지 패턴이 있고 해결 방법도 비슷합니다. 가장 흔한 에러 중 하나는 "Permission denied" 입니다. 이는 해당 파일이나 폴더에 접근할 권한이 없다는 뜻입니다. 리눅스는 보안을 위해 파일마다 접근 권한을 설정하는데, 시스템 파일을 일반 사용자가 수정하려고 하면 이런 에러가 발생합니다. 해결 방법은 sudo를 명령어 앞에 붙여서 관리자 권한으로 실행하는 것입니다. "No such file or directory" 에러는 지정한 파일이나 폴더가 존재하지 않는다는 의미입니다. 파일 이름을 잘못 입력했거나, 파일이 다른 위치에 있을 때 발생합니다. ls 명령어로 현재 위치의 파일들을 확인하고, pwd 명령어로 현재 위치를 확인해보세요. "Command not found" 에러는 입력한 명령어가 존재하지 않거나 설치되지 않았다는 뜻입니다. 명령어 이름을 잘못 입력했거나, 해당 프로그램이 설치되지 않았을 때 발생합니다. 패키지 관리자를 사용해서 필요한 프로그램을 설치하면 해결됩니다. 네트워크 관련 에러도 자주 발생합니다. "Connection refused"는 연결하려는 서버가 응답하지 않는다는 뜻이고, "Network unreachable"은 네트워크 연결 자체에 문제가 있다는 의미입니다. 이런 경우에는 인터넷 연결 상태를 확인하고, 방화벽 설정을 점검해야 합니다. 에러가 발생했을 때는 당황하지 말고 에러 메시지를 자세히 읽어보세요. 대부분의 에러 메시지는 문제의 원인과 해결 방향을 알려줍니다. 구글에서 에러 메시지를 검색하면 비슷한 문제를 겪은 사람들의 해결 방법을 찾을 수 있습니다.
도메인이란 무엇인가요?
도메인은 인터넷상의 주소를 사람이 기억하기 쉽게 만든 이름입니다. 실제로는 모든 웹사이트가 192.168.1.1 같은 숫자로 된 IP 주소를 가지고 있지만, 이런 숫자를 외우기는 매우 어렵습니다. 그래서 siliod.com, google.com처럼 의미 있는 이름을 사용하는 것이 도메인입니다. 도메인은 계층 구조로 되어 있습니다. www.google.com을 예로 들면, com이 최상위 도메인이고, google이 2차 도메인, www가 하위 도메인입니다. 마치 주소에서 나라, 시, 구, 동의 순서와 비슷합니다. 이런 계층 구조 덕분에 전 세계의 수많은 웹사이트들을 체계적으로 관리할 수 있습니다. 도메인을 사용하려면 도메인 등록 업체에서 구매해야 합니다. 도메인은 연간 사용료를 지불하고 임대하는 형태입니다. 인기 있는 도메인 이름은 비싸기도 하고, 이미 다른 사람이 사용하고 있으면 구매할 수 없습니다. 도메인을 구매한 후에는 DNS 설정을 통해 실제 서버의 IP 주소와 연결해야 합니다. 도메인의 가장 큰 장점은 기억하기 쉽다는 것입니다. 숫자로 된 IP 주소 대신 의미 있는 이름을 사용할 수 있어서 브랜딩에도 도움이 됩니다. 또한 서버를 다른 곳으로 이전하더라도 도메인 이름은 그대로 유지할 수 있어서 사용자들이 계속 같은 주소로 접속할 수 있습니다. siliod 같은 서비스에서 도메인을 무료로 제공한다는 것은 매우 큰 장점입니다. 일반적으로 도메인은 연간 1만원에서 수만원의 비용이 들기 때문에, 특히 개인 개발자나 스타트업에게는 초기 비용 부담을 줄여주는 소중한 혜택입니다. 무료 도메인을 통해 더 쉽게 웹 프로젝트를 시작하고 운영할 수 있습니다.
HTTPS의 필요성
HTTPS는 웹사이트와 사용자 사이의 통신을 암호화해서 보호하는 기술입니다. 기존의 HTTP는 정보가 그대로 전송되어서 중간에 누군가 훔쳐볼 수 있지만, HTTPS는 모든 정보를 암호화해서 안전하게 전송합니다. 온라인 쇼핑을 할 때 신용카드 정보를 입력한다고 생각해보세요. 이런 중요한 정보가 암호화되지 않은 채로 전송된다면 해커가 중간에 가로채서 악용할 수 있습니다. HTTPS는 이런 정보를 복잡한 암호로 변환해서 전송하기 때문에, 중간에 가로채더라도 원래 내용을 알 수 없습니다. HTTPS의 또 다른 중요한 역할은 웹사이트의 신원을 확인하는 것입니다. SSL 인증서라는 것을 통해 이 웹사이트가 정말 믿을 수 있는 곳인지 확인할 수 있습니다. 가짜 은행 사이트나 피싱 사이트들은 대부분 정식 인증서를 가지고 있지 않기 때문에, HTTPS를 사용하지 않거나 브라우저에서 경고 메시지를 보여줍니다. 요즘은 구글 같은 검색 엔진에서도 HTTPS를 사용하는 웹사이트를 더 높게 평가합니다. 검색 결과에서 상위에 노출되려면 HTTPS가 거의 필수가 되었습니다. 또한 최신 브라우저들은 HTTPS를 사용하지 않는 웹사이트를 방문할 때 "안전하지 않음"이라는 경고를 표시합니다. 과거에는 SSL 인증서를 구매하고 설정하는 것이 복잡하고 비용이 많이 들었습니다. 하지만 지금은 Let's Encrypt 같은 무료 인증서 서비스가 있어서 누구나 쉽게 HTTPS를 사용할 수 있습니다. siliod 같은 서비스에서 인증서를 자동으로 발급해주는 것은 정말 편리한 기능입니다. 개발자가 복잡한 인증서 설정 과정을 거치지 않고도 안전한 웹사이트를 만들 수 있기 때문입니다. 이런 자동화 덕분에 더 많은 웹사이트들이 HTTPS를 사용하게 되고, 전체적인 인터넷 보안이 향상됩니다.
웹서버란 무엇인가요?
웹서버는 인터넷상에서 웹사이트를 제공하는 컴퓨터 프로그램입니다. 마치 도서관의 사서처럼, 사용자가 웹페이지를 요청하면 해당 파일을 찾아서 전달해주는 역할을 합니다. 우리가 브라우저에 주소를 입력하고 엔터를 누르면, 그 순간 웹서버에게 "이 페이지를 보여주세요"라고 요청을 보내는 것입니다. 웹서버는 이 요청을 받아서 해당하는 HTML 파일, 이미지, CSS 파일 등을 찾아서 우리의 브라우저로 전송합니다. 그러면 브라우저가 이 파일들을 받아서 우리가 보는 웹페이지로 만들어 보여줍니다. 웹서버에는 여러 종류가 있습니다. 가장 유명한 것은 Apache와 Nginx입니다. Apache는 오래되고 안정적인 웹서버로 많은 기능을 제공합니다. Nginx는 상대적으로 새롭지만 빠른 속도와 적은 메모리 사용량으로 인기가 높습니다. 이 외에도 Microsoft의 IIS, 구글의 Tomcat 등 다양한 웹서버들이 있습니다. 웹서버가 하는 일은 단순히 파일을 전달하는 것만이 아닙니다. 사용자 인증을 확인하고, 로그를 기록하고, 보안을 관리하고, 여러 사용자의 요청을 동시에 처리합니다. 또한 PHP나 Python 같은 프로그래밍 언어로 작성된 동적 웹페이지를 실행해서 결과를 전달하기도 합니다. 리눅스는 웹서버를 운영하기에 매우 적합한 운영체제입니다. 안정성이 뛰어나고 보안이 강하며, 무료로 사용할 수 있기 때문입니다. 전 세계 웹사이트의 대부분이 리눅스 서버에서 운영되고 있습니다. 웹서버를 설치하고 관리하는 것도 리눅스에서는 패키지 관리자를 통해 매우 간단합니다. sudo apt install apache2 한 줄로 웹서버를 설치할 수 있고, 바로 웹사이트를 운영할 수 있습니다. 웹서버를 이해하면 인터넷이 어떻게 작동하는지, 웹사이트가 어떻게 만들어지는지 알 수 있습니다. 리눅스와 웹서버는 현대 인터넷의 핵심 기술이며, 이를 배우면 웹 개발과 서버 관리의 기초를 다질 수 있습니다.
Siliod에서 443포트를 사용하는 법
Siliod에서는 인스턴스에 연결할 때 기본적으로 443 포트를 사용합니다.
따라서 HTTPS 등을 위해 443 포트를 사용하려면, 인스턴스 연결 포트를 8443 등으로 변경해야 합니다.

443 포트: HTTPS(HTTP Secure)의 표준 포트입니다.
웹 브라우저가 SSL/TLS로 암호화된 연결을 위해 기본적으로 사용하는 포트입니다.
시스템 포트(1~1023)이므로 사용 시 sudo 권한이 필요합니다.

인스턴스 연결 포트 수정 방법

1.1 GUI인 경우 start.sh 수정
기존 설정의 포트를 8443으로 변경합니다:
/home/ubuntu/.novnc/start.sh 를 열고 마지막 줄 마지막 옵션을 수정하세요
기존 설정:
..... --listen 443
변경된 설정:
..... --listen 8443

1.2 CLI인 경우 Crontab 설정 수정
기존 설정의 포트를 8443으로 변경합니다:
crontab -e
기존 설정:
@reboot sudo /home/ubuntu/.ttyd/build/ttyd --port 443 --ssl .....
변경된 설정:
@reboot sudo /home/ubuntu/.ttyd/build/ttyd --port 8443 --ssl .....

2. 대시보드에서 포트 열기
단계별로 진행합니다:
- 인스턴스 선택: 해당 인스턴스를 클릭하여 선택
- 하단 더보기 메뉴: 인스턴스 상세 정보 하단의 "더보기" 버튼 클릭
- 포트 설정: 네트워크/보안 설정 메뉴로 이동

TCP 8443 포트 추가:
- 프로토콜: TCP
- 포트: 8443
- 소스: 0.0.0.0/0 (모든 IP 허용) 또는 필요에 따라 제한

3. 인스턴스 재시작
설정이 완료되면 인스턴스를 재시작합니다:
- 대시보드에서:
    - 인스턴스 선택
    - "인스턴스 재시작" 버튼 클릭

- 또는 터미널에서:
sudo reboot

4. 접속 확인
재시작 후 다음 URL로 접속하여 연결을 확인합니다:
https://{instance-id}.siliod.com:8443

HTTPS 서버를 만들고 싶다면
Siliod에서는 인스턴스를 생성할 때 도메인과 TLS 인증서가 자동으로 생성되므로, 별도로 인증서를 다시 발급받을 필요가 없습니다.

SSL(TLS) 인증서 경로:
- 인증서 파일: /etc/letsencrypt/live/{instance-id}.siliod.com/fullchain.pem
- 개인키 파일: /etc/letsencrypt/live/{instance-id}.siliod.com/privkey.pem

인증서 파일 확인 명령어:
sudo ls -la /etc/letsencrypt/live/{instance-id}.siliod.com/
스토리지 용량을 늘리고 싶어요
1. 스토리지 용량 추가
단계별로 진행합니다:
- 인스턴스 선택: 해당 인스턴스를 클릭하여 선택
- 하단 더보기 메뉴: 인스턴스 상세 정보 하단의 "더보기" 버튼 클릭
- 스토리지 용량: 원하는 만큼 용량을 늘린후 "용량 추가" 버튼 클릭 (용량 줄이기는 불가능 합니다.)

2. 스토리지 리사이징
리눅스에서는 하드웨어 용량을 늘려도 바로 용량을 사용 할수 없습니다.

먼저 df -h로 기존 스토리지 용량을 확인후
sudo growpart /dev/nvme0n1 1
sudo resize2fs /dev/nvme0n1p1
를 입력해 스토리지 리사이징 후

다시 df -h로 변경된 스토리지 용량을 확인하세요
